
# PaceMaker 프로젝트 통합 개발 프롬프트

---

## 1. 프로젝트 개요

**프로젝트명:** PaceMaker: 데이터 기반 수학 모의고사 맞춤형 분석 및 학습 전략 솔루션

**목적:**
학생들의 수학 모의고사 풀이 과정을 체계적으로 디지털화하여, 개인별 학습 상황을 종합적으로 분석하고, 실질적인 시험 전략과 학습 계획을 데이터 기반으로 제공하는 애플리케이션을 개발한다.

**핵심 문제 인식:**
기존 수학 학습 프로그램은 모의고사 연습 중 문항별 응답 시간을 개별적으로 측정, 비교하여 학습자의 집중도나 전반적 학습 상태를 면밀히 분석하는 데 한계가 있었다. 학생들은 최종 점수, 오답 수, 총 소요 시간 등 단편적인 결과 데이터만으로 간접적으로 자신의 실력을 추론해야 했으며, 어떤 유형의 문제를 쉽게 풀었고 어떤 문제에서 어려움을 겪었는지 기억에만 의존해야 했다.

## 2. 문제 데이터베이스 구축

### 2.1 출처
- **한국교육과정평가원(KICE) 기출문제**를 모은 PDF 파일을 수집하여 문제 데이터베이스를 구축한다. (PDF 파일은 디렉토리 안에 존재)
- 수능, 6월/9월 모의평가 등 평가원 공식 출제 문항을 대상으로 한다.

### 2.2 PDF 수집 및 파싱 파이프라인

```
[PDF 수집] → [메타데이터 태깅] → [DB 저장]
```

| 단계 | 설명 |
|------|------|
| **PDF 수집** | 평가원 기출문제 PDF 파일을 연도별, 시험 유형별로 수집(디렉토리 안에 존재, 문항별 이미지 수동으로 추출하여 폴더 구축함) |
| **메타데이터 태깅** | 각 문항에 아래 메타데이터를 부여('연도-과목-시험유형-문항번호-단원' 형식으로 이미지 파일명에 기재) |

### 2.3 문항 데이터 스키마

```json
{
  "question_id": "25-대수-09-22-로그함수",
  "year": 2024,
  "subject_area": "대수 | 확통 | 미적분1",
  "exam_type": "수능 | 10월 학력평가 | 9월 모의평가 | 7월 학력평가 | 6월 모의평가 | 5월 학력평가 | 3월 학력평가",
  "question_number": 22,
  "unit": "수열의 극한 | 미분법 | 적분법 | 확률 | 통계 | 이차곡선 | 벡터 ...",
  "question_image": "path/to/image.png",
  "question_type": "객관식(5지선다) | 주관식(단답형)",
  "answer": "정답값",
  "average_correct_rate": 0.35,
  "average_solving_time_sec": null,
}
```

### 2.4 데이터 수집 시 유의사항
- 기출문제의 정답, 정답률 등은 공개 데이터를 활용한다.
- 단원(unit) 분류는 현행 교육과정 체계에 맞추어 태깅한다.
- 연도별, 시험별로 체계적으로 분류하여 사용자가 원하는 범위의 문제를 선택할 수 있도록 한다.

## 3. 앱 세션 구조 및 흐름

PaceMaker는 **진단 → 시험 → 리뷰 → 리포트**의 4단계 세션으로 구성된다.

### 3.1 Diagnostic Session (진단 세션)
- **대상:** 최초 이용자
- **구성:** DB에서 무작위 선별된 20문항
- **수집 데이터:** 문항별 응답 시간, 정오답 여부
- **목적:** 사용자의 baseline(기준선) 데이터 형성

### 3.2 Exam Session (시험 세션)
- **대상:** 기존 사용 기록이 있는 사용자
- **구성:** 사용자가 원하는 단원(예: 미적분, 기하 등)과 문항 수를 선택하여 맞춤형 모의고사 진행
- **수집 데이터:** 문항별 응답 시간, 정오답 여부 (백그라운드 실시간 수집)
- **실시간 분석:**
  - 내장된 평균/표준편차 데이터를 활용하여 사용자의 **Z-score** 추정
  - 응답 시간 데이터 기반 **이상치(outlier) 탐지**

### 3.3 Review Session (리뷰 세션)
- 문항 목록과 함께 다음 정보를 직관적으로 제공:
  - 사용자의 정오답 여부
  - 평균 정답률
  - 4가지 상태 인덱스 (Zoning Out, Guessing, Efficient, Unefficient)
- 사용자가 직접 리뷰하며, 실제로 집중력이 떨어졌는지, 무작위로 답을 선택했는지 등을 기록
- **메타인지 향상**을 위한 개인화 데이터로 축적

### 3.4 Report Session (리포트 세션)
- 심층 통계 지표 시각화 제공:
  - 응답 시간 vs 정답률 산점도(Scatter Plot)
  - Z-score 분포
  - 유형별 Efficient/Unefficient 문항 분포
- 향후 시험에서 Zoning Out 및 Guessing을 방지하기 위한 **최적 시간 배분 전략** 제시

## 4. 핵심 통계 분석 모듈

### 4.1 이상치 탐지 (IQR 기반)

```
         ◀──── 정상 범위 ────▶
  ──────┬────┬─────────┬────┬──────
        │ Q1 │   Q2    │ Q3 │
  ──────┴────┴─────────┴────┴──────
  ▲                         ▲
  Q1 - 1.5×IQR              Q3 + 1.5×IQR
  (Guessing Risk)           (Zoning Out Risk)
```

| 분류 | 조건 | 의미 |
|------|------|------|
| **Zoning Out Risk** | 응답 시간 > Q3 + 1.5 × IQR (상한선) | 집중력 저하가 의심되는 문항 |
| **Guessing Risk** | 응답 시간 < Q1 - 1.5 × IQR (하한선) | 찍기(추측)가 의심되는 문항 |

### 4.2 효율성 분석 (상관계수 기반)

| 분류 | 조건 | 의미 |
|------|------|------|
| **Efficient** | 응답 시간과 정답률 간 양의 상관관계 | 시간 투자 대비 효율이 좋은 문항 |
| **Unefficient** | 응답 시간과 정답률 간 음의 상관관계 | 시간을 투자했음에도 오답률이 높은 문항 |

### 4.3 Z-score 산출
- 각 문항에 대한 사용자의 응답 시간을 전체 사용자 평균/표준편차와 비교하여 표준화 점수를 산출
- 유사 실력대 사용자의 평균 데이터와 비교 분석

### 4.4 시간 배분 전략 도출
- 유형별 Efficient/Unefficient 문항 분포를 기반으로, 어떤 문항의 풀이 시간을 줄이고 어떤 어려운 문항에 확보된 시간을 투자해야 하는지 데이터 기반 전략 수립
- 특정 문항에 일정 시간 이상 머물 경우 집중력이 떨어지고 정답률이 감소하는 임계점을 시각적으로 확인 가능
- 문항당 최적 투자 시간에 대한 실질적 가이드라인 제공

## 7. API 엔드포인트 설계 (주요)

| Method | Endpoint | 설명 |
|--------|----------|------|
| `POST` | `/api/users/register` | 사용자 등록 |
| `GET` | `/api/questions/diagnostic` | 진단 세션용 20문항 랜덤 추출 |
| `POST` | `/api/sessions/start` | 시험 세션 시작 (단원, 문항수 선택) |
| `POST` | `/api/responses/submit` | 문항별 응답 제출 (응답시간 포함) |
| `GET` | `/api/sessions/{id}/review` | 리뷰 세션 데이터 조회 |
| `PATCH` | `/api/responses/{id}/self-review` | 사용자 자기진단 피드백 기록 |
| `GET` | `/api/sessions/{id}/report` | 리포트 세션 (통계 분석 결과) |
| `GET` | `/api/users/{id}/history` | 사용자 누적 학습 이력 조회 |
| `GET` | `/api/analysis/time-strategy` | 최적 시간 배분 전략 조회 |

## 8. 리포트 산출물 구성

리포트 세션에서 사용자에게 제공되는 분석 항목은 다음과 같다:

| 항목 | 내용 |
|------|------|
| **문항별 응답 시간 vs 정답률 산점도** | X축: 응답 시간, Y축: 정답률, 상태 인덱스별 색상 구분 |
| **Z-score 분포 차트** | 사용자의 문항별 Z-score를 시각화, 유사 실력대 평균과 비교 |
| **상태 인덱스 요약** | Zoning Out / Guessing / Efficient / Unefficient 문항 목록 및 비율 |
| **단원별 강약점 분석** | 단원별 정답률, 평균 응답 시간을 타 사용자와 비교 |
| **집중력 임계점 분석** | 특정 시간 이상 문항에 머물 경우 정답률이 급감하는 임계점 표시 |
| **최적 시간 배분 전략** | 문항 유형별 권장 풀이 시간 및 시간 재배분 가이드 |
| **누적 추이 분석** | 세션별 성적, 집중력, 효율성 변화 추이 (다회 사용 시) |

## 9. 기대 효과

1. **직관 의존 탈피:** "시간이 부족했다", "특정 문제가 어려웠다" 등의 추상적 분석에서 벗어나, 시험 난이도, 타 사용자의 소요 시간, 시험 기록을 객관적으로 비교하여 과학적 리뷰가 가능하다.

2. **정밀한 강약점 진단:** 문항별 소요 시간과 정답률을 타 사용자 평균 데이터와 교차 분석하여, 어떤 유형에 강하고 어떤 유형에 약한지 명확히 인식할 수 있다.

3. **실전 전략 수립:** 데이터에 엄밀히 기반하여, 어떤 문항의 풀이 시간을 줄이고 확보된 시간을 어떤 고난도 문항에 투자할지에 대한 구체적 전략을 도출한다.

4. **메타인지 향상:** 리뷰 세션에서의 자기 진단 피드백이 개인화 데이터로 축적되어, 학습자의 자기 인식 능력이 점진적으로 향상된다.

## 10. 개발 우선순위 및 마일스톤

| 단계 | 내용 | 우선순위 |
|------|------|----------|
| **Phase 1** | 평가원 기출문제 DB 구축 | 최우선 |
| **Phase 2** | 진단 세션 및 시험 세션 핵심 기능 (타이머, 응답 수집) | 높음 |
| **Phase 3** | 통계 분석 모듈 (IQR 이상치, Z-score, 상관분석) | 높음 |
| **Phase 4** | 리뷰 세션 및 사용자 자기진단 피드백 기능 | 중간 |
| **Phase 5** | 리포트 세션 시각화 및 시간 배분 전략 엔진 | 중간 |
| **Phase 6** | 누적 데이터 기반 추이 분석 및 고도화 | 추후 |
